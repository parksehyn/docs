# 상품 목록 페이지 요구사항 명세서

## 1. 문서 개요
* **프로젝트명:** 쇼핑몰 상품 목록 페이지 구현
* **문서 작성일:** 2024년 00월 00일
* **작성자:** [본인 이름]
* **문서 버전:** v1.0

## 2. 프로젝트 목표
사용자가 상품을 편리하게 탐색하고, 장바구니에 원하는 상품을 담을 수 있는 모바일 반응형 웹 페이지를 구현합니다.

## 3. 핵심 요구사항 (Key Features)

### 3.1 UI/UX 요구사항
* **모바일 중심 UI (Mobile First):** 모바일 환경에 최적화된 레이아웃을 제공해야 합니다.
* **레이아웃:** 상품 목록은 **2개의 컬럼(2-Column Grid)** 형태로 구성하여 한 줄에 두 개의 상품 카드가 보이도록 배치합니다.
* **이미지:** 모든 상품 카드는 **상품 사진**을 포함해야 합니다.

### 3.2 기능적 요구사항 (Functional Requirements)

#### A. 헤더 (Header)
* 페이지 상단에 고정된 헤더가 존재해야 합니다.
* **장바구니 아이콘:** 우측 상단에 장바구니 아이콘을 배치합니다.
* **실시간 카운트:** 장바구니에 담긴 상품의 개수가 아이콘 주변에 배지(Badge) 형태로 표시되어야 하며, 상품을 담을 때마다 **실시간으로 숫자가 업데이트**되어야 합니다.

#### B. 상품 목록 (Product List)
* **상품 카드 정보:** 각 상품 카드는 다음 정보를 포함해야 합니다.
  * 상품 이미지
  * 상품명
  * 가격
  * 담기 버튼
* **담기 버튼 기능:**
  * 사용자가 '담기' 버튼을 클릭하면 장바구니에 상품이 추가됩니다.
  * **상태 변화:** 클릭 시 버튼의 상태가 변경되어야 합니다 (예: 버튼 색상 변경, 텍스트가 '담기 완료'로 변경, 또는 버튼 비활성화 등 사용자가 담았음을 인지할 수 있는 피드백 제공).

### 3.3 비기능적 요구사항 (Non-Functional Requirements)
* **배포 (Deployment):** 결과물은 실제 웹 호스팅 서비스를 통해 배포되어야 하며, URL을 통해 접속 가능해야 합니다.
  * **배포 URL:** [여기에 배포한 URL을 기재하세요. 예: https://my-shop.vercel.app]

## 4. 데이터 요구사항
* 상품 데이터는 JSON 형태의 Mock Data 또는 API를 통해 불러와 렌더링해야 합니다.
* 각 상품 객체는 `id`, `name`, `price`, `imageUrl` 등의 필드를 가져야 합니다.
